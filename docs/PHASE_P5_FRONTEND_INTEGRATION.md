# Phase P.5 – Frontend Integration Layer

## Goals
- **API versioning** – expose the full backend under a versioned prefix (`/api/v1`).
- **Health & readiness** – simple liveness (`GET /health`) and readiness (`GET /ready`) endpoints.
- **Typed client library** – a deterministic Python client (`scrubin.client.ScrubinClient`) that wraps all HTTP calls with typed request/response helpers.
- **Configuration module** – immutable `scrubin.config.Config` reads environment variables (`SCRUBIN_API_URL`, `SCRUBIN_TIMEOUT`).
- **Frontend‑safe DTOs** – request/response payloads are plain JSON‑serialisable structures; the domain engine (`WorldState`, `TimelineEvent`) remains untouched.
- **Deterministic testing** – tests verify that health/readiness endpoints are always reachable, that the client correctly serialises requests, and that environment‑driven configuration loads predictably.

## Implementation Highlights
### API versioning
```python
# scrubin/server/app.py
app.include_router(router)                     # legacy un‑prefixed routes (existing tests)
app.include_router(router, prefix="/api/v1") # new versioned namespace
```
All existing endpoints (`/session/*`, `/sessions`) are therefore available both as
`/session/create` **and** as `/api/v1/session/create`.

### Health & readiness
```python
@router.get("/health")
def health():
    return {"status": "ok"}

@router.get("/ready")
def ready():
    return {"status": "ready"}
```
Both routes are also reachable under the `/api/v1` prefix.

### Configuration (`scrubin/config.py`)
```python
@dataclass(frozen=True, slots=True)
class Config:
    api_base_url: str = field(default_factory=lambda: os.getenv("SCRUBIN_API_URL", "http://localhost"))
    timeout: int = field(default_factory=lambda: int(os.getenv("SCRUBIN_TIMEOUT", "30")))
```
Environment variables are read at import time; the dataclass is immutable, ensuring deterministic behaviour.

### Typed client (`scrubin/client.py`)
* Built on **httpx** – works with a live HTTP server or, for tests, with the FastAPI
  ASGI app via `httpx.ASGITransport`.
* All public methods accept an optional ``token`` which is turned into a
  ``Bearer`` ``Authorization`` header, keeping authentication handling simple.
* Responses are returned as plain ``dict`` objects; callers can further map them
  to Pydantic models if desired.

### Frontend‑safe DTOs
The client never touches the internal `WorldState` objects; it treats them as
opaque JSON payloads (`dict`). This guarantees that the deterministic simulation
engine remains completely isolated from frontend concerns.

## Deterministic Tests (`tests/test_frontend_integration.py`)
1. **Health** – asserts `{"status": "ok"}`.
2. **Readiness** – asserts `{"status": "ready"}`.
3. **API client** – end‑to‑end flow using the client (create, action, save,
   load, delete, list) confirming that each step succeeds and that the final
   state is as expected.
4. **Configuration loading** – monkey‑patches environment variables and checks
   that `Config` reflects them exactly.

All tests run against the in‑process FastAPI ``TestClient`` via the ASGI
transport, keeping execution fast (< 1 s) and fully deterministic.
